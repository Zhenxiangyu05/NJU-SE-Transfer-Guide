# 面试经验

## 面试形式

从 **2025 年** 开始，软件学院转专业面试采用较为标准化的流程。

### 2025 年面试形式

2025 年面试时间较短，每位同学严格控制在 **5 分钟** 左右，主要包括：

- **2 分钟自我介绍**
- **2 个问题回答**
  - 其中一个问题由考生抽取
  - 另一个问题由老师指定
- **全程录像**
- **5 位评委老师面试 1 位学生**
- 设有一定追问环节

### 2026 年面试形式

2026 年面试流程进一步标准化。现场在一个大会议室中进行，对面有10位评委老师，右侧设有计时器，题目会出现在右侧大屏幕上，坐下准备好后开始计时。

面试主要包括：

- **2 分钟自我介绍**
- **4 个问题回答**
- 问题方向包括：
  - 软件工程与软件设计
  - 程序设计与编程细节
  - 计算系统基础
  - 思政与科技伦理
- 共 **10 位评委老师**
- 基本无追问环节
- 全程需要自行把控时间

需要特别注意的是，老师一般不会主动提醒你每一题用了多久。因此，正式面试时一定要注意时间分配，不要在某一个问题上停留太久，否则容易影响后续问题的回答节奏。

---

## 面试准备建议

面试前一定要提前练习，尤其是 **2 分钟自我介绍**。

建议重点准备以下内容：

- 为什么想转入软件工程专业；
- 自己有哪些编程经历或项目经历；
- 对软件工程的理解；
- 机试和算法学习经历；
- 是否有印象深刻的 Debug / 重构 / 项目开发经历；
- 常见软件工程概念；
- C / Java 基础知识；
- 计算系统基础相关问题；
- AI、开源、科技伦理、社会责任等开放性问题。

面试不是单纯考察背诵能力，而是考察你是否具备进入软件工程专业学习的基本素质，包括：

- 表达能力
- 逻辑组织能力
- 工程思维
- 代码规范意识
- 问题分析能力
- 对专业的真实兴趣和持续投入

# 2025 年面试真题整理

> 以下内容根据个人回忆及同学交流整理，仅供参考。

## 一、软件工程与代码质量

1. 如果你的代码写得一团糟，但是能够运行，你会不会对它进行重构？你觉得重构的意义是什么？

2. 软件工程的定义是什么？你认为软件工程是否仅仅等于写代码？

3. 什么是好代码？如何让自己的代码变得更好？

4. 设计一个图书管理系统，请从面向对象编程或结构化编程的角度进行分析。

5. 做过哪些项目？在项目中如何处理类与类之间的关系？

---

## 二、测试、Debug 与工程实践

6. 程序测试的概念是什么？具体应该如何进行测试？

7. 测试和 Debug 的本质区别是什么？如果测试跑出了一个偶发性的 Bug，但你自己在本地怎么 Debug 都无法复现，你会怎么排查？

8. 你有没有印象深刻的 Debug 经历？平常是如何定位和修改 Bug 的？

9. 经历过痛苦的 Debug 之后，你在写代码时会做哪些预见性的工作来减少 Bug 的产生？

---

## 三、程序设计与面向对象

10. 函数（Function）和方法（Method）究竟是什么？它们有什么用？

11. 在面向对象中，方法的重载（Overloading）和重写（Overriding）有什么区别？

12. 面向对象有哪些基本特点？请解释封装、继承和多态。

13. 如何学习一门新的编程语言？

---

## 四、项目经历与个人经历

14. 有没有特殊的一次大项目经历？遇到了什么困难？

15. 你最痛苦的一次写代码经历是什么？

---

## 五、软件过程与系统基础

16. 什么是瀑布模型？对于学生团队开发项目而言，更适合使用瀑布模型还是敏捷开发？为什么？

17. 什么是 JVM 垃圾回收机制？

18. 什么是冯诺依曼结构？

---

# 2026 年面试真题整理

> 以下内容根据 2026 年转专业面试情况整理，仅供参考。

## 大一篇

### 一、转专业动机与个人认知

1. 为什么想转入软件工程专业？

2. 你对软件工程的兴趣是从哪里来的？

3. 你认为自己目前有哪些不足？在未来的软件工程学习过程中，如何利用自己的跨专业背景弥补这些不足？

4. 第一次让你眼前一亮的算法或数据结构是什么？

---

### 二、软件工程与软件设计

5. 如果让你设计一个学生成绩管理系统，你会怎么设计？

6. 设计出一个登录系统后，你会如何设计测试场景？

7. 如果要阅读一个 5000 行的代码项目，你会怎么读？

8. Git 是什么？你有没有用过 Git？如果没有，说说其他类似工具。Git 有哪些常用操作命令？

9. 大模型已经可以写代码了，未来还需要软件工程师吗？

---

### 三、程序设计与语言思维

10. 什么是未定义行为（Undefined Behavior, UB）？

11. 为什么 C 语言规范不强制对未定义行为报错，或者给出一个确定的结果？

12. 你有没有遇到过或写出过未定义行为？

13. 如果由你来设计一门编程语言，你会保留未定义行为吗？

14. 你怎么看待 `i`、`flag`、`data` 这类极简变量命名？

15. 在 C 语言中，函数名该不该使用缩写？

16. 如果让你重写 C 标准库，你会把标准库函数名加长吗？

---

### 四、C 语言与内存基础

17. 全局变量和局部变量分别存储在内存中的什么位置？

18. 如果没有赋初始值，例如 `int a;`，全局变量和局部变量分别会有什么表现？

19. 栈和堆有什么区别？

20. 全局变量和局部变量分别应该在什么情况下使用？

21. 阅读下面的代码，指出每个变量的存储位置，以及它们的值是什么：

```c
int a;
int b = 1;

int main() {
    int c;
    int d = 2;
    char* s = "hello";
}
```

---

### 五、计算系统基础

22. 为什么计算机要使用二进制？

23. 如果由你来设计计算机的数字系统，你会选择几进制？

24. 为什么整数运算会发生溢出，而逻辑运算不会？

25. 如果让你在计算机中设计一种表示小数的编码方式，你会怎么设计？

26. 对于 3A 大作而言，到底是显卡更重要，还是 CPU 更重要？

27. 计算机的核心数量是不是越多越好？

28. 如果语言不通，该如何与外星人交流？

29. 人机交互的理想状态是什么？

30. 一枚质地均匀的硬币和一比特表示的信息量是否一样？

---

### 六、科技伦理与社会责任

31. 青年科技人才应该如何助力乡村振兴？

32. 在设计数字系统或政府网站时，如何兼顾效率指标与包容性，例如适老化和残障人士使用需求？

33. 为什么开源不等于无国界？请从个人未来职业规划的角度出发，谈谈如何在开源项目、开源社区等方面为祖国做贡献。

34. 目前互联网平台采用的推荐算法可能导致“信息茧房”现象。有些工程师认为应该采用其他算法，更多考虑社会责任，但这样可能导致平台效益下降。你认为应该如何权衡？请联系社会主义核心价值观，谈谈互联网平台在信息传播中的主体作用。

35. 设计一个数字系统时，如何平衡效率和包容性？如何考虑老年人和残障人士？应该秉持什么理念？请至少提出一条兼顾包容性和效率的方案。

36. 科技人才如何助力实现乡村振兴？

37. 青少年应该如何正确对待网络舆论？

## 大二篇

> 以下问题根据 2026 年大二转专业面试情况整理。

### 1. AI 辅助开发与项目归属

如果老师让你做一个系统，你几乎全程使用 AI，几天就完成了。这样的项目还能算是你自己做的吗？它与完全由自己完成的项目有什么区别？

---

### 2. AI 与开发者能力

现在有些人在 Debug 时，会直接把报错信息发给 AI 进行修复。你认为这种方式是好事还是坏事？它是否会削弱开发者的独立思考能力？

---

### 3. 程序员角色变化

在 AI 编程能力快速提升的背景下，你如何理解“程序员”这一职业的角色变化？未来程序员是否可能被 AI 取代？

---

### 4. AI 部署方式选择

如果厂商提出两种 AI 部署方案：

- 一种是本地（手机端）部署模型
- 一种是云端服务器调用模型

你会如何选择？请从性能、成本、隐私与延迟等角度分析。

---

### 5. 软件系统退化问题分析

手机使用时间久后，可能出现：

- App 启动变慢
- 系统卡顿
- 电池性能下降

你认为这些问题主要来自硬件老化、软件复杂度增加，还是其他因素？请分析原因。

---

### 6. 开源代码使用与工程伦理

在使用 GitHub 开源代码或他人代码时：

- 是否必须标注来源？为什么？
- 如果未标注但未用于盈利，你会如何处理？
- 如果在项目中发现队友使用开源代码但未注明来源，你会如何应对？提醒、内部沟通，还是其他方式？

---

### 7. 团队协作与责任划分

学校组织一个团队开发小程序。组员 A 负责前端，但在临近验收前一天晚上出现了重大 Bug，且 A 因为有事没有时间修复。负责后端的组员 B 擅自修改了 A 的代码试图修复，结果引入了更多 Bug，并导致验收发布会出现严重问题。

请回答：

- 小 A、小 B 和其他组员分别有什么责任？
- 如果你是组长，你会如何处理？
- 代码能力和团队责任哪个更重要？
- 如果你是组长，事后组织复盘会时，你会说什么、做什么？

## 备考建议

从 2025 和 2026 年的面试题可以看出，软件学院转专业面试并不是单纯考察记忆，而是更重视综合能力。

建议重点准备以下几个方向：

1. **自我介绍和转专业动机**  
   要能清楚说明为什么想转入软件工程，以及自己为此做了哪些准备。

2. **编程经历和项目经历**  
   尽量准备 1～2 个真实项目或编程经历，能够说清楚遇到的问题、解决过程和收获。

3. **软件工程基本概念**  
   包括软件工程定义、代码质量、重构、测试、Debug、面向对象、开发模型等。

4. **计算机基础知识**  
   包括 C 语言内存模型、栈和堆、冯诺依曼结构、二进制、小数编码、整数溢出等。

5. **AI 与工程伦理问题**  
   包括 AI 辅助编程、开源规范、推荐算法、数字包容性、科技人才社会责任等。

6. **表达与控时能力**  
   2026 年面试问题较多，时间较紧，建议提前进行模拟面试，训练在有限时间内清晰表达核心观点的能力。

面试前不要只背答案，更要理解每个问题背后的考察点。老师真正想看的不是你是否背出了某个标准定义，而是你是否具备持续学习、独立思考和解决复杂问题的能力。
