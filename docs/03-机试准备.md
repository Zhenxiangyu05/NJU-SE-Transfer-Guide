# 机试准备

## 机试概况

在软件工程转专业考核中，**机试是最重要的考核环节之一。**

2026 年之前，机试成绩作为综合考核的一部分，按一定比例计入最终成绩。

自 **2026 年** 起，软件学院调整了考核方式：

- **机试满分 100 分；**
- **成绩达到 60 分及以上，方可进入面试环节；**
- **未达到 60 分者，将失去本次转专业后续考核资格。**

因此，机试不仅影响最终录取结果，更直接决定你是否能够进入下一轮面试考核。

---

## 考试形式（2026 年参考）

- **考试平台：** SEECODER
- **编程语言：** Java
- **考试时长：** 120 分钟
- **题目数量：** 5 道
- **题目来源：** 力扣（LeetCode）原题或改编题、《软件工程与计算 I》机考类型综合题

SEECODER 同时也是《软件工程与计算 I》课程使用的在线评测平台，因此建议提前熟悉平台的基本操作、代码提交方式以及相关开发环境。

---

## 考前注意事项

**一定要提前到达考场，确认考试环境正常。**

建议重点检查以下内容：

- SEECODER 是否能够正常登录；
- Git 是否能够正常使用；
- 网络和开发环境是否正常；
- 是否能够顺利下载试卷并提交代码。

这是我亲身踩过的坑。

考试当天，由于 Git 环境出现问题，我花费了约 **15 分钟** 才成功下载试卷。虽然最终完成了考试，但这件事确实对心态产生了一定影响。

因此，建议大家尽可能提前到场，预留充足时间检查设备、网络和开发环境，避免因为非技术因素影响考试发挥。

---

## 我的建议

如果你目前还是**大一新生**，且没有计算机学习经历，建议首先认真学习 **C语言程序设计基础（CPL）** 课程，扎实打好编程基础。

在学有余力的情况下，可以提前接触 Java，熟悉 Java 的基本语法和面向对象编程思想。

大一上学期结束后，建议尽早开始系统学习 **Java 语言**。

虽然算法能力才是决定机试成绩的核心，但如果考试时还需要花费大量时间处理 Java 语法细节，或者因为语言不熟悉而影响做题效率，就会非常可惜。

因此，我建议将 **Java 学习** 和 **算法训练** 同步推进，为机试做好充分准备。

## 如何提升算法能力

机试的核心仍然是算法能力。

在算法学习方面，我非常推荐大家跟随 GitHub 上的经典项目：

📚 [代码随想录](https://github.com/youngyangyang04/leetcode-master)

这也是我自己的主要学习路径。

《代码随想录》的优点在于：它不是简单地罗列题目，而是按照知识点和难度循序渐进地组织内容。认真学完之后，可以逐步建立起比较完整的算法知识体系。

对于转专业机试，我建议重点掌握以下章节：

- 数组
- 链表
- 哈希表
- 字符串
- 双指针
- 栈与队列
- 贪心算法
- 单调栈
- 动态规划

这些内容都是机试中非常重要的基础题型和高频考点。

如果时间和能力允许，可以进一步学习：

- 二叉树
- 回溯算法
- 图论基础

我个人认为，如果能够比较扎实地掌握上述重点章节，应对软件工程转专业机试会更有底气。

---

## 为什么推荐《代码随想录》

推荐《代码随想录》并不是因为它“题量多”，而是因为它的学习顺序和知识体系非常适合从零开始建立算法能力。

从往年情况来看，转专业机试中也确实出现过与《代码随想录》题单高度相关的题目。例如：

- **2025 年转专业机试**：出现过《代码随想录》贪心算法篇中的相关题目 —— 力扣 55「跳跃游戏」。
- **2026 年转专业机试**：出现过《代码随想录》字符串篇中的相关题目 —— 力扣 151「反转字符串中的单词」。

当然，不能简单地认为“刷完代码随想录就一定能押中原题”。更重要的是，通过这些题目训练自己的算法思维、代码实现能力和临场调试能力。

如果你能够认真完成《代码随想录》中的重点章节，不仅对转专业机试有帮助，也会对之后的数据结构与算法学习、实习面试以及求职中的手撕代码产生长期收益。

## 其他推荐题单

除了《代码随想录》之外，也可以补充刷以下经典题单：

- LeetCode Hot 100
- 面试经典 150 题

这两个题单覆盖了大量高频算法题，质量很高，适合作为阶段性巩固和查漏补缺。

不过需要注意的是，机试准备阶段不要一味追求刷题数量。相比“刷了多少题”，更重要的是：

- 是否真正理解题目的核心思路；
- 是否能够独立写出代码；
- 是否能够处理边界情况；
- 是否能够在有限时间内完成调试和提交。

算法学习不是短期突击就能完全掌握的内容，建议尽早开始，持续练习。

## 考前模拟与复习

在考试临近阶段，建议进行**限时训练**，尽可能高强度地模拟真实考试环境。

例如，可以按照正式机试的要求，设置 **120 分钟**，连续完成 5 道题，并在规定时间内完成代码编写、调试和提交。这样可以提前适应考试节奏，训练时间分配能力，也能减少正式考试时的紧张感。

此外，做过的题不要只做一遍。对于重点题型和曾经做错的题目，建议定期回顾和复盘：

- 重新梳理解题思路；
- 总结常见边界情况；
- 复现关键代码模板；
- 记录自己容易犯的错误；
- 尝试在不看题解的情况下重新写出完整代码。

真正有效的刷题不是“看懂题解”，而是能够在考场环境下，独立、稳定、快速地写出正确代码。

因此，考前最后一段时间，与其盲目追求新题数量，不如把已经学过的重点题型反复巩固，把常见算法思想和代码实现练熟。

## 近年真题整理

> 以下内容根据个人回忆及同学交流整理，仅供参考。不同年份题目可能会有变化，请不要将其视为唯一备考依据。

### 2025 年

| 序号 | 题目 | 类型 |
| :---: | :--- | :--- |
| 1 | [力扣 13：罗马数字转整数](https://leetcode.cn/problems/roman-to-integer/description/) | 字符串 / 哈希表 |
| 2 | [力扣 495：提莫攻击](https://leetcode.cn/problems/teemo-attacking/description/) | 数组 / 模拟 |
| 3 | [力扣 55：跳跃游戏](https://leetcode.cn/problems/jump-game/description/) | 贪心算法 |
| 4 | [力扣 228：汇总区间](https://leetcode.cn/problems/summary-ranges/description/) | 数组 / 区间 |
| 5 | 字符表拼写判定题 | 哈希表 / 字符统计 |

其中，第 5 题大致题意如下：

给定一个字符表和一个需要拼写的字符串。字符表中包含 `a-z`、`A-Z` 以及 `*`，其中 `*` 可以替代任意字母；需要拼写的字符串中不会出现 `*`。字符表中的每个字符只能使用一次，判断能否用给定字符表拼出目标字符串。

| 序号 | 题目 | 类型 |
| :---: | :--- | :--- |
| 1 | [力扣 151：反转字符串中的单词](https://leetcode.cn/problems/reverse-words-in-a-string/description/) | 字符串 |

从近两年题目可以看出，转专业机试并不只考察“是否见过原题”，更考察对基础算法思想和代码实现能力的掌握程度。

高频考点主要集中在：

- 字符串处理；
- 数组与哈希表；
- 双指针；
- 贪心算法；
- 动态规划；
- 链表操作；
- 前缀和等基础技巧。

因此，备考时既要重视经典题单，也要重视基础代码能力和临场实现能力。不要只停留在“看懂题解”，而要尽量做到能够在限时条件下独立写出正确代码。
