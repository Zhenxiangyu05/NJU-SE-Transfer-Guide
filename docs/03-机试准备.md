# 机试准备

## 机试概况

在软件工程转专业考核中，**机试是最重要的考核环节之一。**

2026 年之前，机试成绩作为综合考核的一部分，按一定比例计入最终成绩。

 **2026 年** ，软件学院调整了考核方式：

- **机试满分 100 分；**
- **成绩达到 60 分及以上，方可进入面试环节；**
- **未达到 60 分者，将直接失去转专业资格。**

因此，机试不仅决定你的最终成绩，更决定你是否能够进入下一轮考核。

---

## 考试形式（2026 年参考）

- **考试平台：** SEECODER
- **编程语言：** Java
- **考试时长：** 120 分钟
- **题目数量：** 5 道
- **题目来源：** 力扣（LeetCode）原题或改编题、《软件工程与计算I》机考类型综合题

SEECODER 同时也是《软件工程与计算 I》课程使用的在线评测平台，因此建议提前熟悉平台的基本操作和提交流程。

---

## 考前注意事项

**一定要提前到达考场，确认考试环境正常。**

建议重点检查以下内容：

- SEECoder 是否能够正常登录；
- Git 是否能够正常使用；
- 是否能够正常下载试卷；
- 是否能够正常提交代码；
- 网络及开发环境是否正常。

这是我亲身踩过的坑。

考试当天，由于 Git 环境出现问题，我花费了约 **15 分钟** 才成功下载试卷。虽然最终完成了考试，但对心态产生了一定影响。因此，建议大家尽可能提前到场，预留充足时间检查设备和环境，避免因为非技术因素影响发挥。

---

## 我的建议

如果你目前还是**大一新生**，且没有计算机学习经历，建议首先认真学习 **C语言程序设计基础（CPL）** 课程，扎实基础。

在学有余力的情况下，可以提前接触 Java，熟悉基本语法和面向对象编程思想。

大一上学期结束，**务必开始Java语言学习**。

虽然算法能力才是决定机试成绩的核心，但如果考试时还需要花费大量时间处理 Java 语法细节，或者因为语言不熟悉而影响做题效率，就会非常可惜。

因此，我建议将 **Java 学习** 和 **算法训练** 同步推进，为机试做好充分准备。
